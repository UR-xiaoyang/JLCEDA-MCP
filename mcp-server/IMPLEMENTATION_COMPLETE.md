## 🏗️ 技术架构

### 主从架构图

```
┌─────────────────────────────────────────────────────────────┐
│                     多客户端架构                              │
│                                                              │
│  实例 #1 (主服务器)         实例 #2 (客户端)    实例 #3 (客户端) │
│       │                          │                  │        │
│   stdio (JSON-RPC)          stdio (JSON-RPC)   stdio (JSON-RPC)
│       │                          │                  │        │
│       ▼                          ▼                  ▼        │
│  ┌─────────┐             ┌─────────┐         ┌─────────┐   │
│  │  Main   │◄────────────┤ Client  │◄────────┤ Client  │   │
│  │ Server  │ WebSocket   │  Mode   │WebSocket│  Mode   │   │
│  └────┬────┘             └─────────┘         └─────────┘   │
│       │                                                      │
│       │ WebSocket (端口 8765)                               │
│       ▼                                                      │
│  ┌─────────────────────────────────────────────────────┐   │
│  │              EDA 插件 (嘉立创 EDA)                   │   │
│  └─────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────┘
```

### 连接路径

- **EDA 插件** → `ws://127.0.0.1:8765/bridge/ws`
- **内部客户端** → `ws://127.0.0.1:8765/mcp-internal`

### 启动流程

```
实例启动
   ↓
检测端口 8765
   ↓
   ├─→ 可用：启动为主服务器
   │   ├─ 监听 EDA 插件连接
   │   └─ 监听 MCP 客户端连接
   │
   └─→ 被占用：启动为客户端模式
       ├─ 连接到主服务器
       └─ 通过主服务器转发请求
```
