# OpenCode MCP 测试说明

## ℹ️ OpenCode不显示MCP日志

OpenCode不会显示MCP服务器的stderr/stdout输出，这是正常的。

---

## ✅ 如何验证MCP服务器正常

### 方法1：手动测试（已完成）

我们已经手动测试了MCP服务器，如果输出包含：
```
JLCEDA MCP Server v2.0.0
WebSocket server listening on ws://127.0.0.1:8765/bridge/ws
MCP Server started successfully
```

说明MCP服务器可以正常启动。

### 方法2：通过功能测试

**最可靠的验证方法是测试实际功能：**

```
1. 确保OpenCode已打开
   ↓
2. 安装EDA插件并打开原理图
   ↓
3. EDA显示：✅ 已连接到MCP服务器
   ↓
4. 在OpenCode对话："读取当前原理图"
   ↓
5. 如果AI能返回结果 → MCP服务器工作正常 ✅
```

---

## 🔍 如何检查MCP连接状态

### 在EDA中检查

```
菜单：MCP Bridge → 连接状态

✅ 应该显示：
• 服务器运行中
• 活动连接: 1 个
```

### 在OpenCode中测试

```
直接对话测试：
"读取当前原理图"
"搜索STM32"

如果AI能响应 → 说明整个链路正常 ✅
```

## ⚠️ 如果遇到问题

### 问题1：EDA未显示连接提示

**排查：**
1. 查看EDA插件调试日志：`MCP Bridge → 查看调试日志`
2. 查找是否有错误信息

### 问题2：工具调用失败

**排查：**
1. 确认EDA插件显示"已连接"
2. 查看EDA调试日志
3. 确认在原理图/PCB页面

---

**重点：功能正常就说明一切正常，不用担心看不到MCP日志。** ✅
