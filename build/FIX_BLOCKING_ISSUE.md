# 🔧 修复"编程工具打不开"问题

## ⚠️ 问题分析

**现象：** 安装EDA插件后，编程工具（EDA）打不开

**可能原因：**
1. ❌ 插件启动时立即尝试连接，阻塞了EDA主线程
2. ❌ 同步API调用阻塞启动流程
3. ❌ 定时器立即启动占用资源

---

## ✅ 修复措施

### 1. 延迟启动（3秒）

**修改前：**
```typescript
export function startBridgeRuntime(): void {
  // 立即启动页面检查和连接
  startPageCheck();
}
```

**修改后：**
```typescript
function startPageCheck(): void {
  // 延迟3秒启动，避免阻塞EDA启动
  globalThis.setTimeout(() => {
    // 启动定时器和连接
  }, 3000);
}
```

### 2. 非阻塞连接

**修改前：**
```typescript
public async start(): Promise<void> {
  await this.connect(); // 阻塞等待连接成功
}
```

**修改后：**
```typescript
public async start(): Promise<void> {
  // 异步连接，不阻塞
  this.connect().catch((error) => {
    // 连接失败会自动重连，不抛出错误
  });
}
```

### 3. 异步启动客户端

**修改前：**
```typescript
transport.start().then(() => {
  debugLog('Client started');
}).catch(() => {
  // ...
});
```

**修改后：**
```typescript
// 直接异步启动，不等待结果
transport.start().catch(() => {
  // 失败会自动重连
});
```

## 🎯 改进效果

| 方面 | 修改前 | 修改后 |
|------|--------|--------|
| EDA启动 | 可能阻塞 ❌ | 不阻塞 ✅ |
| 连接时机 | 立即连接 | 延迟3秒 ✅ |
