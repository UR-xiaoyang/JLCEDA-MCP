## 🔍 使用场景

### 场景1：检查服务器是否启动

**操作：** `MCP Bridge → 连接状态`

**结果：**
- ✅ 显示 "服务器运行中" → 正常
- ❌ 显示 "服务器未启动" → 需要重启EDA

### 场景2：检查MCP客户端是否连接

**操作：** `MCP Bridge → 连接状态`

**结果：**
- ✅ "活动连接: 1 个" → 已连接
- ⚠️ "当前无客户端连接" → OpenCode未启动或连接失败

### 场景3：排查连接问题

**步骤：**
1. 打开 `MCP Bridge → 连接状态`
2. 检查服务器是否运行
3. 检查是否有活动连接
4. 查看连接详情（连接时长、请求数）

### 场景4：验证工具调用

**观察：**
- 每次AI调用工具时，"处理请求"数量会增加
- "空闲时间"会重置
- "总请求数"累计增长

## 📊 连接状态说明

### 服务器状态指标

| 指标 | 说明 |
|------|------|
| **服务地址** | WebSocket服务器地址 |
| **运行时长** | 从启动到现在的时间 |
| **总请求数** | 累计处理的工具调用次数 |
| **活动连接** | 当前连接的MCP客户端数量 |

### 连接详情指标

| 指标 | 说明 |
|------|------|
| **ID** | 连接的唯一标识符 |
| **连接时长** | 该连接建立多久 |
| **空闲时间** | 距离上次请求过了多久 |
| **处理请求** | 该连接发送的请求次数 |

## 🐛 故障排除

### 问题1：连接状态显示"服务器未启动"

**原因：** 插件启动失败

**解决：**
1. 查看EDA控制台错误
2. 检查8765端口是否被占用
3. 重启JLCEDA EDA

### 问题2：连接状态显示"当前无客户端连接"

**原因：** OpenCode未连接

**解决：**
1. 确认OpenCode已启动
2. 检查OpenCode配置是否正确
3. 查看OpenCode输出窗口是否有错误

### 问题3："处理请求"数量不增加

**原因：** 工具调用失败

**解决：**
1. 查看 `MCP Bridge → 查看调试日志`
2. 检查OpenCode输出窗口的错误
3. 确认当前页面类型（原理图/PCB）
