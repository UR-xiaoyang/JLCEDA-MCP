# 🔧 EDA插件 v2.0 调试版本

## 问题分析

**现象：** 在原理图页面点击"重启服务器"，提示"当前不在原理图或PCB页面"

**可能原因：**
1. `eda.sys_Context.getActivePageType()` 返回的值不是 `'sch'` 或 `'pcb'`
2. API调用时机问题（重启时页面状态获取有延迟）

## 🔍 新版本改进

### 调试信息增强

新版本会在重启时显示**当前页面类型**，帮助诊断问题：

```
提示框内容：
━━━━━━━━━━━━━━━━━━━━━━
当前不在原理图或PCB页面

当前页面类型: [实际返回值]

服务器将在打开原理图/PCB时自动启动。
━━━━━━━━━━━━━━━━━━━━━━
```

### 日志增强

添加了详细的调试日志：
- 获取页面类型时输出日志
- 重启检查时输出页面类型
- 失败时输出错误信息

## 📋 测试步骤

### 步骤1：安装新版本

文件：`D:\jlc-assistant\JLCEDA-MCP\build\jlceda-mcp-bridge-2.0.0.eext`

### 步骤2：打开原理图并测试

1. 打开原理图文件
2. 确认看到提示：`MCP Bridge服务器已启动`
3. 点击菜单：`MCP Bridge → 重启服务器`
4. **观察提示框内容**

### 步骤3：查看调试日志

如果仍然提示不在正确页面：

1. 点击菜单：`MCP Bridge → 查看调试日志`
2. 查找类似信息：
   ```
   [Bridge Runtime] Current page type: xxx
   [Bridge Runtime] Restart check - isValid: false, pageType: xxx
   ```

## 🔧 可能的页面类型值

根据JLCEDA EDA API，可能返回的值：

| 返回值 | 说明 |
|--------|------|
| `'sch'` | 原理图页面 ✅ |
| `'pcb'` | PCB页面 ✅ |
| `'home'` | 主页面 |
| `'lib'` | 库管理页面 |
| `'schlib'` | 原理图库 |
| `'pcblib'` | PCB库 |
| 其他 | 未知页面类型 |

## 🔄 临时解决方案

如果重启功能异常，可以使用以下方法：

```
1. 切换到Home页面
2. 再切换回原理图
3. 服务器会自动重启
```

### 方案2：重启EDA
```
完全重启JLCEDA EDA
```
