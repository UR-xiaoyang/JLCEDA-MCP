## 🔄 重启服务器功能

### 何时使用

**适用场景：**
- ⚠️ 连接异常，无法恢复
- ⚠️ OpenCode显示连接超时
- ⚠️ 工具调用一直失败
- ⚠️ 端口被占用后释放

**操作步骤：**
```
1. 菜单：MCP Bridge → 重启服务器
   ↓
2. 看到提示：正在重启MCP Bridge服务器...
   ↓
3. 等待500ms
   ↓
4. 如在原理图/PCB页面 → 自动重启成功
5. 如在其他页面 → 提示打开原理图/PCB后自动启动
```

### 重启行为

**在原理图/PCB页面：**
```
点击重启
  ↓
停止当前服务器（断开所有连接）
  ↓
等待500ms
  ↓
重新启动服务器
  ↓
✅ 重启完成，可重新连接
```

**在其他页面（Home等）：**
```
点击重启
  ↓
显示提示：当前不在原理图或PCB页面
  ↓
说明：服务器将在打开原理图/PCB时自动启动
```

## 🔍 故障排除

### 问题1：服务器未启动

**检查：** `MCP Bridge → 连接状态`

**原因：**
- ❌ 当前在Home页面

**解决：**
- ✅ 打开原理图或PCB文件

---

### 问题2：连接超时

**检查：** OpenCode输出窗口

**原因：**
- ⚠️ 服务器未启动
- ⚠️ 端口被占用
- ⚠️ 防火墙阻止

**解决：**
1. 检查服务器状态
2. 检查端口：`netstat -ano | findstr 8765`
3. 尝试重启：`MCP Bridge → 重启服务器`

---

### 问题3：工具调用失败

**检查：** `MCP Bridge → 查看调试日志`

**原因：**
- ⚠️ Handler执行错误
- ⚠️ 页面类型不匹配

**解决：**
1. 查看调试日志排查具体错误
2. 确认当前在原理图/PCB页面
3. 尝试重启服务器

---

### 问题4：连接突然断开

**现象：** OpenCode显示连接断开

**原因：**
- ⚠️ 切换到了非原理图/PCB页面
- ⚠️ 服务器异常崩溃

**解决：**
1. 确认当前页面类型
2. 如在正确页面，尝试重启服务器
3. 重启OpenCode
